Dark Mode no Tails

A partir do Tails 6.0 (2024-02-27) passou-se a utilizar como base o Debian 12 que por sua vez utiliza o Gnome 43, implementando por padrão a função dark mode. Alguns comandos abaixos estão inutilizáveis para versões mais recentes do Tails, porém não irei remover o tópico deste site.

Por padrão o Tails não fornece a opção de dark mode, mas pelo fato dele usar o gnome com sua interface gráfica, podemos alterar para torná-lo mais confortável, especialmente ao navegar à noite

Para habilitar o tema escuro no Tails, digite os seguintes comandos no Terminal:

  • amnesia@amnesia:~$
    • gsettings set org.gnome.desktop.interface gtk-theme 'Adwaita-dark'
    • gsettings set org.gnome.desktop.background picture-options 'none'
    • gsettings set org.gnome.desktop.background primary-color '#000000'
    • gsettings set org.gnome.desktop.background secondary-color '#000000'

Em seguida, para tornar o tema escuro persistente, digite também os seguintes comandos:

  • amnesia@amnesia:~$
    • cd /live/persistence/TailsData_unlocked/dotfiles
    • mkdir.config
    • mkdir .config/dconf
    • mkdir .config/gtk-3.0
    • cp ~/.config/dconf/user .config/dconf/

Então você tem que criar um arquivo de texto:

  • amnesia@amnesia:~$
    • cd /live/persistence/TailsData_unlocked/dotfiles
    • nano .config/gtk-3.0/settings.ini

Em seguida, digite as seguintes linhas no arquivo de texto:

  • amnesia@amnesia:~$ 
    • [Sttings]
gtk-application-prefer-dark-theme = true
gtk-theme-name = Adwaita-dark

Salve o arquivo de texto e saia do editor de texto pressionando CTRL+X, depois Y (ou S) e depois pressione ENTER.

Agora você terminou: reinicie o Tails e aproveite um ambiente com o conforto do tema escuro.

Tor Browser seguro e Persistente

Tails fornece um ambiente seguro. Tudo é totalmente criptografado e qualquer coisa que não seja expressamente salva no diretório Persistent é excluída quando você desliga o sistema. Isso também significa que você também perde todas as configurações da interface do usuário e do software (especialmente o TOR). Ter que configurar tudo toda vez que você inicializa seu PC com o Tails não é apenas cansativo, mas também pode representar uma ameaça à segurança, já que o Nível de Segurança TOR é definido por padrão como Padrão (o mais baixo possível) e você pode esquecer de configurá-lo como Mais Seguro antes para começar a navegar na Web ou na Dark Web. No entanto, você pode salvar alguns arquivos de configuração no diretório Persistent. Assumimos que você criou o espaço Persistente no Tails, se ainda não o fez, você pode criá-lo na tela de inicialização do Tails. É assim que você pode ter mais segurança e conforto no Tails, configurando o TOR para o nível de segurança mais seguro, definindo um tema escuro para a interface do usuário da área de trabalho e o TOR e mantendo-o persistente após a reinicialização.

Em primeiro lugar, configure o TOR conforme explicado na seção dedicada acima. Certifique-se de ter definido o nível de segurança como mais seguro.

Para tornar a configuração do TOR permanente, abra o Terminal primeiro e digite os seguintes comandos:

  • amnesia@amnesia:~$
    • cd /live/persistence/TailsData_unlocked/dotfiles
    • mkdir .tor-browser
    • mkdir .tor-browser/profile.default
    • cp ~/.tor-browser/profile.default/prefs.js .tor-browser/profile.default/
    • chmod 640 .tor-browser/profile.default/prefs.js

Você pode copiar esse comando único logo abaixo:

  • amnesia@amnesia:~$
    • cd /live/persistence/TailsData_unlocked/dotfiles && mkdir .tor-browser && mkdir .tor-browser/profile.default && cp ~/.tor-browser/profile.default/prefs.js .tor-browser/profile.default/ && chmod 640 .tor-browser/profile.default/prefs.js

Iniciando o Dino com Torsocks no Tails

Para começar o processo de inicialização do dino-im através do torsocks no Tails, será necessário que você tenha a partição Persistent.

  • amnesia@amnesia:~$
    • echo -e '/home/amnesia/.local/share/dino source=dino' | sudo tee -a /live/persistence/TailsData_unlocked/persistence.conf > /dev/null
  • Reiniciar Tails
  • amnesia@amnesia:~$
    • sudo apt update
      • A atualização deve terminar sem erros! Às vezes há problemas de conexão com o servidor de atualização debian. Se houver falha na atualização, tente novamente alguns segundos depois.
    • sudo apt install dino-im dino-im-common
      • escolha "Instalar sempre" na GUI, o pacote é salvo no Persistent Storage e será reinstalado automaticamente toda vez que você inicia o Tails. O pacote também será atualizado automaticamente quando você se conectar ao Internet. A instalação de software adicional pode demorar um pouco em inicialização, você será notificado quando a instalação for concluída. Você pode verificar o sucesso clicando no ícone Data/Hora no Tails.
      • espere alguns segundos até " [INFO] Adicionando à lista de pacotes adicionais: {'dino-im'} " aparece em o terminal e, em seguida, pressione ENTER
  • amnesia@amnesia:~$
    • sudo sed -i 's/dino-im$/dino-im\/buster-backports/' /live/persistence/TailsData_unlocked/live-additional-software.conf
      • As seguintes instruções irão criar um script de shell inicial e adicionar um ícone na área de trabalho chamado DinoTails para sua área de trabalho, para que você possa iniciar o DinoTails apenas Clicando. Se você não quiser usar o ícone da área de trabalho, pule estas instruções para iniciar o DinoTails a partir do terminal ou ALT+F2 com "torsocks dino-im".
    • echo -e '#!/bin/bash\n\ntorsocks dino-im' | tee /live/persistence/TailsData_unlocked/dino/DinoTails.sh > /dev/null
    • chmod 700 /live/persistence/TailsData_unlocked/dino/DinoTails.sh
    • echo -e '[Desktop Entry]\nVersion=1.0\nEncoding=UTF-8\nName=DinoTails\nType=Application\nTerminal=false\nExec=/live/persistence/TailsData_unlocked/dino/DinoTails.sh\nIcon=im.dino.Dino\nStartupNotify=true\n' | tee /home/amnesia/Desktop/DinoTails.desktop > /dev/null
    • mkdir -p /live/persistence/TailsData_unlocked/dotfiles/Desktop
    • cp -r /home/amnesia/Desktop/DinoTails.desktop /live/persistence/TailsData_unlocked/dotfiles/Desktop/DinoTails.desktop

    = = = = = = = = = = = = = = = = = = = = OU = = = = = = = = = = = = = = = = = = = =

    • sudo sed -i 's/dino-im$/dino-im\/buster-backports/' /live/persistence/TailsData_unlocked/live-additional-software.conf && echo -e '#!/bin/bash\n\ntorsocks dino-im' | tee /live/persistence/TailsData_unlocked/dino/DinoTails.sh > /dev/null && chmod 700 /live/persistence/TailsData_unlocked/dino/DinoTails.sh && echo -e '[Desktop Entry]\nVersion=1.0\nEncoding=UTF-8\nName=DinoTails\nType=Application\nTerminal=false\nExec=/live/persistence/TailsData_unlocked/dino/DinoTails.sh\nIcon=im.dino.Dino\nStartupNotify=true\n' | tee /home/amnesia/Desktop/DinoTails.desktop > /dev/null && mkdir -p /live/persistence/TailsData_unlocked/dotfiles/Desktop && cp -r /home/amnesia/Desktop/DinoTails.desktop /live/persistence/TailsData_unlocked/dotfiles/Desktop/DinoTails.desktop

  • Reiniciar Tails
    • Note que ao reiniciar o Tails, o dino não será instalado e irá aparcer erro na instalação automática.
  • amnesia@amnesia:~$
    • sudo apt install dino-im dino-im-common
      • "Programas Adicionais"
        • Após a instalação manual do dino, iremos abrir o aplicativo "Programas Adicionais" e iremos remover o dino-im-commom e o dino-im/buster-backports.
    • Gerenciador de pacotes Synaptic
      • Iremos abrir o Gerenciador de pacotes Synaptic e iremos atualizar todos os pacotes, logo após a atualização de todos os pacotes, podemos reiniciar o Tails.
  • Reiniciar Tails
  • Gerenciador de pacotes Synaptic
    • Após ter reiniciado o Tails e com todos os pacotes atualizados, iremos abrir novamente do synaptic e iremos pesquisar por dino-im, iremos marcar esse pacote e também o pacote dino-im-common (caso ele não esteja marcado por padrão), logo em seguida iremos baixar esses dois pacotes e desta vez, ao finalizar a instalação, podemos clicar em Instalar sempre na notificação que irá aparecer.
    • Ao realizar esse procedimento, o dino será instalado 'automágicamente' assim que iniciarmos o Tails, sem nenhuma mensagem de erro ou falhas.
  • Clique com o botão direito do mouse no DinoTails-Icon na área de trabalho
    • escolha Permitir lançamento, repita isso uma vez após cada reinicialização
  • Clique duas vezes no DinoTails-Icon para iniciar Dino com Torsocks
    • se você iniciar o Dino a partir do menu de aplicativos "Aplicativos -> Internet -> Dino", não vai usar Torsocks
      • isso não será uma segurança problema, simplesmente não funcionará porque as conexões serão negadas por firewall
  • Sua conta Dino configurações, chaves, contatos, conversas, arquivos e avatares serão armazenado em /live/persistence/TailsData_unlocked/dino/ agora você pode usar persistentemente Dino no Tails apenas por (Permitir Iniciar) e clicar no ícone DinoTails Desktop

Guia básico de GnuPG

GNU Privacy Guard é um software livre alternativo ao conjunto de softwares criptográficos PGP da Symantec, e em conformidade com o RFC 4880, a especificação de rastreio de padrões IETF do OpenPGP. Versões modernas do PGP possuem interoperabilidade com o GnuPG e com outros sistemas compatíveis com o OpenPGP.

  • Passo 1 (Comando para iniciar a criação da chave):

    • Abaixo da versão 2 do gpg, utilizamos o comando: gpg --gen-key
    • Em versões acima da 2, utilizamos o comando: gpg --full-generate-key

  • Passo 2 (Procedimentos para a criação da chave):

    • Em seguida ele irá pedir o tipo de chave, ecolheremos o default RSA and RSA, e logo após iremos esolher a quantidade de bits desejado. Escolha o máximo possível, quanto maior, melhor. Logo após selecionarmos, temos que definir o tempo para a expiração da chave, ou para nunca expirar.

  • Passo 3 (Informações pessoais):

    • Após concluido os procedimentos anteriores, você pecisará definir alguma informações "pessoais", sendo elas: nome, endereço de E-Mail e um comentário (opcional). Logo após você definir as informações, ele irá pedir uma senha para a chave, que será usada para criptografar e descriptografar documentos. Recomendo uma senha bastante segura.

  • Chave criada, agora precisamos aprender alguns comandos básicos.

    • gpg --list-key ou gpg -k para listar as chaves públicas salva em seu PC
    • gpg --list-secret-keys ou gpg -K para listar suas chaves privadas salva em seu PC
    • gpg --fingerprint para listar em blocos de 4 dígitos o código das chaves salva em seu PC

      • OBS: seu key id é o código da sua chave ou os ultimos 16 dígitos do código de sua chave ou o seu E-Mail.

  • Para exportar sua chave pública:
    • gpg --export --armor SEU_KEY_ID > NOME_DO_DOCUMENTO
      • Ex: gpg --export --armor 1234567823456789 > public-key.txt
  • Para exportar a sua chave privada:
    • gpg --export-secret-keys --armor SEU_KEY_ID > NOME_DO_DOCUMENTO
      • Ex: gpg --export-secret-keys --armor 1234567823456789 > private-key.txt
        • Note que ao exportar sua chave privada, será necessário uma senha. Essa senha será a que você definiu na criação de sua chave, essa senha também será usada caso você queira importar a sua chave (privada) e também será necessário para descriptografar arquivos.
  • OBS: recomendo que deixe guardado como backup em algum dispositivo de armazenamento a sua chave privada, de preferência, obviamente, de forma criptografada. Você pode criptografar a sua chave de forma simétrica usando o comando a baixo, será necessário uma senha para criptografar e descriptografar o arquivo da sua chave privada:
    • gpg --symmetric NOME_DO_ARQUIVO
      • Ex: gpg --symmetric private-key.txt
    • Note que esse comando acima serve para encriptar qualquer tipo de arquivo, obviamente, não apenas a sua chave privada. E a lógica de utilização é bem simples, usamos o parâmetro gpg --symmetric NOME_DO_ARQUIVO como você acabou de ver e para descriptografar esse arquivo encriptado de forma simétrica com gpg, iremos usar o comando abaixo:
    • gpg -d ARQUIVO_CRIPTOGRAFADO > ARQUIVO_DESCRIPTOGRAFADO
      • Ex: gpg -d private-key.txt.gpg > private-key.txt
  • Para importar uma chave pública de alguém, podemos usar o seguinte comando:
    • gpg --import NOME_DO_ARQUIVO_DA_CHAVE
      • Ex: gpg --import public-key.txt
        • Note que para importar sua chave privada ou sua chave pública, iremos seguir a mesma lógica do gpg --import para importar qualquer tipo de chave.
  • Para criptografar qualquer arquivo, podemos usar o seguinte comando:
    • gpg --encrypt --recipient KEY_ID NOME_DO_ARQUIVO
      • Ex: Irei criptografar um texto (texto.txt) para o BOB, eu já importei a chave pública dele e tenho em mãos o seu key id, usarei como key id do BOB o seu E-Mail (contact@bob.org). Então o comando ficará assim:
        • gpg --encrypt --recipient bob@contact.org texto.txt
  • Para descriptografar um arquivo:
    • gpg --decrypt NOME_DO_ARQUIVO_CRIPTOGRAFADO > NOME_PARA_O_ARQUIVO
      • Ex: Recebi um arquivo criptografado de Alice com minha chave pública, e vou descriptografar. O arquivo recebido é (imagem.png.gpg). Então o comando ficará assim:
        • gpg --decrypt imagem.png.gpg > imagem.png
  • Para remover uma chave GnuPG do contatos de seu dispositivo, usaremos o comando:
    • gpg --delete-keys KEY_ID
      • Ex: gpg --delete-keys A8EK5B96B041D193
  • Caso a chave a ser deletada também seja uma private key em seu contato. Usaremos o comando:
    • gpg --delete-secret-keys KEY_ID
      • Ex: gpg --delete-secret-keys A8EK5B96B041D193

Um puco sobre Hash

Uma função hash é um algoritmo que mapeia dados de comprimento variável para dados de comprimento fixo. Os valores retornados por uma função hash são chamados valores hash, códigos hash, somas hash, checksums ou simplesmente hashes.

Gerando um HASH

Para gerar um HASH é bem simples, vamos lá. Para começar precisamos saber qual função hash iremos utilizar, como: MD5, SHA-1, SHA-2, SHA-512, RIPEMD-160 e etc. No meu caso, eu irei usar o sha256sum.

Dentro de uma pasta eu tenho cinco imagens do Tails.

amnesia@amnesia:~$ ls
tails-amd64-5.3.1.img    tails-amd64-5.4.img    tails-amd64-5.5.img    tails-amd64-5.6.img    tails-amd64-5.7.img

Eu irei criar um hash apenas do arquivo tails-amd64-5.4.img, nesse caso o comando ficará asism:

  • amnesia@amnesia:~$ sha256sum tails-amd64-5.4.img

Note que nesse caso o hash foi "cuspido na tela" desta forma:

amnesia@amnesia:~$ sha256sum tails-amd64-5.4.img
f72be910c7edd29b111679bceb6d76b08fbc41ac434073b0bb1ae103f9320e95 tails-amd64-5.4.img

Para isso, devemos redirecionar a saida padrão para dentro de um arquivo de texto. Então o comando ficará assim:

  • amnesia@amnesia:~$ sha256sum tails-amd64-5.4.img > sha256sum.txt

Após a finalização do processo, se dermos um ls nesse diretório, iremos perceber que temos um arquivo chamado sha256sum.txt e dentro deste arquivo teremos o nosso hash. Desta forma você aprendeu a como gerar um hash de um arquivo e redirecioná-lo para um arquivo de texto. Porém queremos gerar o hash de todos os arquivos dentro deste diretório, então o comando ficará assim:

  • amnesia@amnesia:~$ sha256sum  tails-amd64-5.3.1.img  tails-amd64-5.4.img  tails-amd64-5.5.img  tails-amd64-5.6.img  tails-amd64-5.7.img > sha256sum.txt

Se você chegou até aqui, você percebeu que no arquivo sha256sum.txt gerado através do comando acima só tem o hash do ultimo arquivo, no caso, o tails-amd64-5.7.img. Isso acontece porque usamos o ">" para redirecionar a saida padrão para um arquivo, porém desta forma ele sempre irá sobreescrever o arquivo anterio, ex: quando executamos esse comando ele vai gerar o hash do primeiro arquivo da sequência em que listamos, que no caso será o tails-amd64-5.3.1.img e o resultado do hash gerado será redirecionado para dentro do arquivo sha256sum.txt e quando o resultado do hash do segundo arquivo listado for gerado e redirecionado para o arquivo de texto, ele irá sobreescrever o hash anterior. Desta forma só ficará o hash do ultimo arquivo listado dentro do bloco de texto. Para resolvermos isso, basta substituir o ">" por ">>", desta forma ele irá concatenar, ou seja, a cada modificação que for necessário para inserir algum dado dentro deste bloco de texto, será inserido uma linha abaixo sem sobreescrever os dados anteriores.

O comando ficará assim:

  • amnesia@amnesia:~$ sha256sum  tails-amd64-5.3.1.img  tails-amd64-5.4.img  tails-amd64-5.5.img  tails-amd64-5.6.img  tails-amd64-5.7.img >> sha256sum.txt

Desta forma se dermos um cat no nosso arquivo de texto, teremos esse resultado:

        amnesia@amnesia:~$ cat sha256sum.txt
        2b5910a061f097b1990f886e2e59c25e609fda582d000d54c29446e02db5c654  tails-amd64-5.3.1.img
        f72be910c7edd29b111679bceb6d76b08fbc41ac434073b0bb1ae103f9320e95  tails-amd64-5.4.img
        a38f8ed073831eae95a52d9cc3315f2b367e1663d42d3870f1f5a068ad33bcdb  tails-amd64-5.5.img
        ae91b531cacdac6d4ae0d0436442fefc52a8df6dff3e704b3ec2b4d00228dcef  tails-amd64-5.6.img
        82fd7c1f8162a2b299970e4bd3f7cc0fea5fb29e71f370150339ce9c2d9e3411  tails-amd64-5.7.img

Sei que ninguém merece ter que digitar todos os arquivos dentro de um diretório manualmente. Imagina se você tem 50 arquivos num diretório? Para isso podemos usar um atalho e o nosso comando ficará assim: sha256sum * >> sha256sum.txt
O " * " significa todos, desta forma ele irá listar automaticamente todos os arquivos naquele diretório. Ex:

  • amnesia@amnesia:~$ sha256sum * >> sha256sum.txt

verificando as somas de hash

Para verificarmos as somas de hash é bem simples, basta digitar o segunte comando:

amnesia@amnesia:~$ sha256sum --ignore-missing --check sha256sum.txt
